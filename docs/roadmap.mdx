# Roadmap


- Golang bases connector support (Full-sync & incremental) - V0

- Parallelism support - V2

- Horizontal scaling (If required) - V2
    - Kafka kind of connectors will definitely need it
    - Helps in protecting 1 stream sync from others

- Lake based optimisations :
    - When to trigger compactions (We will maintain metadata of all data being ingested to better help us predict when to compact) - V2
    - Near realtime sync - 1 min benchmark V0
    - GCP & Azure support - V2
    - Multiple catalog support - V2

- Client defined partition - V1

- Full refresh specific table. - V1

- New table full sync - V1
