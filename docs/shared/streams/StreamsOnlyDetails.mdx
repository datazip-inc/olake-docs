| **Component**                    | **Data Type** | **Example Value**                                | **Description & Possible Values**                                                                                                                                               |
|----------------------------------|---------------|--------------------------------------------------|---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| **`name`**                       | string        | `"stream_8"`                                     | Unique identifier for the stream. Each stream must have a unique name.                                                                                                          |
| **`namespace`**                  | string        | `"olake_db"`                                     | The grouping or database name that the stream belongs to. Helps organize streams by logical or physical data sources.                                                           |
| **`type_schema`**                | object        | *(JSON object with properties)*                  | Defines the structure of the records in the stream. Contains a `properties` object that maps each field (key) to its allowed data types (e.g., string, integer, array, object). |
| **`supported_sync_modes`**       | array         | `["full_refresh", "cdc", "incremental"]`         | Lists the synchronization modes the stream supports. Typically includes `"full_refresh"`, `"cdc"`, and `"incremental"`.                                                         |
| **`source_defined_primary_key`** | array         | `["_id"]`                                        | Specifies the field(s) that uniquely identify a record within the stream. Ensures data uniqueness and integrity.                                                                |
| **`available_cursor_fields`**    | array         | `["_id", "name", "marks", "updated_at"]`         | Lists fields that can be used to track synchronization progress in `incremental` sync mode.                                                                                     |
| **`sync_mode`**                  | string        | `"incremental"`                                  | Indicates the active synchronization mode. Possible values are `"cdc"` (change data capture), `"incremental"`, or `"full_refresh"`.                                             |
| **`cursor_field`**               | string        | `"updated_at"`                                   | Defines the cursor field used to track incremental sync. A secondary cursor field can also be specified, separated by a colon.                                                  |
