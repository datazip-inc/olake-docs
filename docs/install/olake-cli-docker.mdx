# OLake CLI using Docker

OLake CLI can be installed and executed using the official Docker images. Each supported source driver has a dedicated image available on the [OLake DockerHub page](https://hub.docker.com/u/olakego).

## Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed and running on the host system
- Access to the internet to pull Docker images

## Configuration and Setup


**1. Navigate to the OLake DockerHub Repository** <br/>
Go to [OLake DockerHub repository](https://hub.docker.com/u/olakego) to browse the available source driver images.
    <div style={{textAlign: 'center'}}>
    ![Docker source images](/img/docs/install/docker-cli/docker-source-images.png)
    </div>
**2. Select the Source Driver** <br/>
Choose the correct source driver from the table below. Use the corresponding value in place of `[SOURCE-NAME]` in the Docker command.
<div style={{ display: "flex", justifyContent: "center", textAlign: "center"}}>

| Driver Name | [SOURCE-NAME] |
|-------------|---------------|
| Postgres    | `postgres`    |
| MySQL       | `mysql`       |
| Oracle      | `oracle`      |
| MongoDB     | `mongodb`     |
</div>
**3. Pull the Docker Image** <br/>
Pull the Docker image for your chosen source driver. Ensure that the Docker daemon is running on your machine.
    ```
    docker pull olakego/source-[SOURCE-NAME]:dev-latest
    ```
    <div style={{textAlign: 'center'}}>
    ![Docker image pull command](/img/docs/install/docker-cli/docker-source-pull.png)
    </div>



## Next Steps
After the initial configuration is completed:

1. Create a local folder to store configuration files such as `source.json` and `destination.json`.  
2. Refer to the driver-specific documentation for detailed configuration options and commands.
