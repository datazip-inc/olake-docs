# OLake commands and flags

OLake supports multiple commands, each serving different use cases.  
This document covers all commands available in the OLake CLI as of [v0.2.0](https://github.com/datazip-inc/olake/releases/tag/v0.2.0).


Commands can be run directly using the `build.sh` script, the generated binary, or through the [OLake Docker CLI](/docs/install/docker-cli).

```bash title="build.sh command"
./build.sh driver-[SOURCE-NAME] [COMMAND] [FLAG]
```

In the above command:
- Replace SOURCE-NAME with the desired source driver.
- Select the command and flags from the options provided in the sections below.

:::info Using the binary
After running the build.sh command, a binary file is generated in the configuration directory. This binary can be used directly to execute commands without invoking build.sh.
:::

## Commands

| command  | description                                                                                                                                 | additional notes                                                                                                                          |
| -------- | ------------------------------------------------------------------------------------------------------------------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------- |
| check    | Checks the connection to the source or destination.                                                                                    | <ul><li>Use the [`--config`](/docs/community/commands-and-flags#2-config) flag for the source.</li><li>Use the [`--destination`](/docs/community/commands-and-flags#4-destination) flag for the destination.</li></ul>                                        |
| spec     | Generates JSON Schema and UI Schema (used to render and validate with [RJSF](https://rjsf-team.github.io/react-jsonschema-form/docs/) ) | <ul><li> **Optional:** Use the [`--destination-type`](/docs/community/commands-and-flags#8-destination-type) flag to generate the destination spec.</li></ul>                                                |
| discover |   Generates a streams.json file containing information about all streams in the source.                                         | <ul><li>Use the [`--config`](/docs/community/commands-and-flags#2-config) flag.</li><li>**Optional:** Use the [`--destination-database-prefix`](/docs/community/commands-and-flags#7-destination-database-prefix) and [`--streams`](/docs/community/commands-and-flags#3-streams) flags.</li></ul>                         |
| sync     | Sync data from the source to the destination.                                                                                              | <ul><li>use [`--config`](/docs/community/commands-and-flags#2-config), [`--streams`](/docs/community/commands-and-flags#3-streams) and [`--destination`](/docs/community/commands-and-flags#4-destination) flags.</li><li>**optional:** use [`--state`](/docs/community/commands-and-flags#5-state) flag to run sync with state.</li></ul> |

## Flags

### 1. Help
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --help
```
**Description:**
- Displays all available commands and flags supported by the current OLake CLI version.
- The shorthand -h can also be used.


### 2. Config
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --config [PATH_TO_CONFIG_FILE]
```
**Description:** 
- Specifies the path to the source configuration file.
- For details about configuration files for different sources, see:
    - [MongoDB Config file](/docs/connectors/mongodb#2-provide-configuration-details-1)
    - [MySQL Config file](/docs/connectors/mysql#2-provide-configuration-details-1)
    - [Oracle Config file](/docs/connectors/oracle#2-provide-configuration-details-1)
    - [PostgreSQL Config file](/docs/connectors/postgres#2-provide-configuration-details-1)

### 3. Streams
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --streams [PATH_TO_STREAMS_FILE]
```
**Description:** 
- Specifies the path to the streams.json file. This file is generated after the discover command.
- If the `--streams` flag is provided during discovery, the existing streams.json is updated by preserving prior user changes and adding new ones detected in the source database.

To know more about `streams.json`, read [Streams Config](/docs/install/docker-cli#streams-config).

### 4. Destination
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --destination [PATH_TO_DESTINATION_FILE]
```
**Description:**
- Specifies the path to the destination.json file.
- For details about destination configuration files, see:
    - [AWS Glue Catalog configuration](/docs/writers/iceberg/catalog/glue#configuration)
    - [REST Catalog configuration](/docs/writers/iceberg/catalog/rest)
    - [JDBC Catalog configuration](/docs/writers/iceberg/catalog/jdbc#configuration)
    - [Hive Catalog configuration](/docs/writers/iceberg/catalog/hive#configuration)
    - [Parquet configuration](/docs/writers/parquet/config#configuration)

### 5. State 
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --state [PATH_TO_STATE_FILE]
```
**Description:**  
- Specifies the path to the state file. The state file is required during the sync command and stores metadata used to resume full refresh, CDC, or incremental sync.

### 6. Clear Destination
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --clear-destination
``` 
**Description:** 
- Clears the destination and resets the state file only for the selected streams in `streams.json`.

### 7. Destination database prefix
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --destination-database-prefix 
```
**Description:** 
- Sets a prefix for the database name created in the destination.
- For example :  With prefix `olake`, the database name appears as `olake_mydb` in the destination.

### 8. Destination type
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --destination-type [TYPE_OF_DESTINATION]
```
**Description:** 
- Used with the `spec` command to generate a specification for the destination.
- `TYPE_OF_DESTINATION` can be any OLake supported destination, for example: iceberg or parquet.

### 9. Decryption of configuration files
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --encryption-key [DECRYPTION_KEY]
```
**Description:** 
- Accepts a key that OLake uses to decrypt encrypted configuration files when running commands. 
- Supported values include KMS keys, UUIDs, or custom strings.

### 10. No Save
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --no-save
```
**Description:** 
- Flag to skip logging artifacts in file.

### 11. Timeout
```
./build.sh driver-[SOURCE-NAME] [COMMAND] --timeout [TIMEOUT_IN_SECONDS]
```
**Description:** 
- Overrides the default timeout value. 
- Applicable only to the `discover` command.
