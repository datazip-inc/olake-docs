# What are Catalogs?

A **catalog** is the place where OLake finds and manages your Iceberg tables. It stores metadata such as table names, schemas, and file locations. Before you can run optimization on tables, OLake must know which catalog (and database) those tables belong to.

In the OLake UI, you work with catalogs from the **optimization** section. There are two types: **OLake Managed Catalogs** and **External Catalogs**.

## OLake Managed Catalogs

**OLake Managed Catalogs** are created automatically when you use OLake Ingestion to write data to an Iceberg destination.

- Whenever you configure an ingestion job with an Iceberg destination (for example, AWS Glue, REST catalog, Hive Metastore, or JDBC catalog), OLake registers that destination as a catalog under **optimization → Catalogs**.
- You do not need to add these catalogs manually. They appear as soon as the corresponding ingestion destination exists.
- Tables that are being replicated by OLake Ingestion to that destination are part of this managed catalog. You can run optimization on those tables directly from the **Tables** page by selecting the catalog and database.

If you already have at least one ingestion job writing to Iceberg, you will see the matching **OLake Managed Catalog** on the Catalogs page and can skip adding a catalog for optimization unless you want to optimize tables from another source.

## External Catalogs

**External Catalogs** are catalogs that you add yourself by clicking **New Catalog** on the **Catalogs** page. They represent Iceberg catalogs for which **no table is currently ingested by OLake**.

Use an external catalog when:

- You want to run optimization on Iceberg tables that were created outside OLake (for example, by Spark, Flink, or another tool).
- The same catalog is used by other systems and you want OLake to compact those tables without replicating data via Ingestion.

After you add an external catalog and connect it, you can select it (and a database) on the **Tables** page to see and optimize the tables in that catalog.

## How to Add a Catalog

You add a catalog from the **Catalogs** page under the optimization section. Use this when you need an **External Catalog**. If you already have an OLake Managed Catalog for your destination, you can skip this.

1. In the OLake UI sidebar, open the **optimization** dropdown and go to the **Catalogs** page.
2. Select **New Catalog**.

![Add catalog view](/img/docs/getting-started/configure-your-first-optimization/add-catalog.webp)

3. Enter the catalog details (for example, catalog name, type, and connection settings).
4. Select **Connect** to save and validate the catalog.

![Catalog connected view](/img/docs/getting-started/configure-your-first-optimization/connect-catalog.webp)

Once the catalog is connected, it appears in the **Select Catalog** dropdown on the **Tables** page. Select it (and then a database) to list and configure optimization for tables in that catalog.

For a full walkthrough that includes adding a catalog and configuring your first optimization job, see [Configure Your First Optimization](/docs/getting-started/configure-first-optimization).
